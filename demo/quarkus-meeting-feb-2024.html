OpenRewrite

Migration Swiss knife toolbox

What is OpenRewrite?

  • Huge collection of recipes = Java classes executed through the visitor pattern
  • Can be used to transform OR analyze
  • Supports dryrun or run mode
  • Executed using Maven

BUT... Limitations

  • Slow when several mvn commands must be executed
  • No easy way to design unit tests
  • No API or Library

Quarkus Rewrite Client

Client and Library designed around OpenRewrite

Bypasses most of the limitations described previously

Installation

Compile locally or install using jbang


mvn clean install -DskipTests -Dquarkus.package.type=uber-jar
jbang app install --force --catalog jbang-catalog-dev.json openrewrite

Demo 1 Rewrite Client


pushd test-project/simple

# Dry run (default)
openrewrite . -r org.openrewrite.java.format.AutoFormat

# Run (apply changes)
openrewrite . -r org.openrewrite.java.format.AutoFormat -d false

# Using a rewrite.yml config
openrewrite . -c rewrite.yml
popd

# Or specify path directly
openrewrite test-project/simple -c rewrite.yml

What about Migrating Applications?

The Challenge

  • OpenRewrite tool ⇒ perfect to execute YAML recipes
  • Scenarios can be really complex even when migrating some REST annotations

Demo 3 Complex Migration


pushd rewrite-client/test-project/quarkus-resteasy-classic-app
openrewrite . -c rewrite.yml

Pain Points

  • YAML format is quite verbose, users will make errors
  • Users should be educated, trained, and ideally supported by a migration tool

The mtool Project

Quarkus Client able to analyze, generate a migration plan, and transform

Demo 4 Migration Scenario

Spring Boot Todo → Quarkus Todo

  • Show 1-2 rules: preconditions and when condition
  • Life becomes easier as Antlr can parse/process the Query

Demo 4 Run Analysis


# Analyze with precondition rules
mtool analyze ./applications/spring-boot-todo-app \
  -r ./cookbook/rules/quarkus-spring-precondition

# Analyze with full rules
mtool analyze ./applications/spring-boot-todo-app \
  -r ./cookbook/rules/quarkus-spring

A report = migration plan is generated, packaging instructions :-)

Demo 4 Spring Boot Todo App


# Show that the Spring Boot Todo app is working
mvn clean package spring-boot:run
firefox localhost:8081

Demo 4 Transform


# Dry run - generates patch files
mtool transform ./applications/spring-boot-todo-app \
  -p openrewrite --dry-run

# Show patch files generated!

# Apply transformation
mtool transform ./applications/spring-boot-todo-app \
  -p openrewrite
